### 2. 타겟 & 시장
| 항목 | 결정 |
|---|---|
| 주 페르소나 | 일본 20~30대 여성 (한국 인바운드 #1 코호트) |
| 확장 순서 | 일본 → 대만·홍콩 → 동남아 |
| 트립 패턴 | 3~4일, 서울 시술 + 지방 축제 양방향 동선 |
| 진입 허브 | 서울 · 부산 · 청주(중심) · 제주 (1년 내 무안 추가) |

### 3. 제품 구조 (3계층 펀넬)
```
Layer 1: 검색 (트래픽)       → K-뷰티 + 축제 + 명인 + 맛집 통합 검색
Layer 2: AI 가이드 (회원화)   → 일정·시술·축제·교통 통합 플래너
Layer 3: 메이트 매칭 (접착)   → 현지 대학생 페어 동행
```

### 4. 플랫폼 DNA — **공급자 셀프 업로드 마켓플레이스**
- 회사가 상품을 만들지 않음. **공급자가 스스로 올림**
- 공급자 타입: 명인·템플스테이·피부과·축제사무국·학교메이트·정원·셰프
- 플랫폼 역할 (2026-04-16 재정의): **연결 · 큐레이션 · AI 심사·승인 · AI 번역 · 일정 조합 · 매칭** (결제·정산·거래 중개는 제외)
- 공급자 거래 수수료 **0%** — 여행자·공급자 간 결제는 공급자 직접 수령 (ACCESS_CONTROL §7)

### 5. 메이트 매칭 룰 (2026-04-15 재설계)
| 항목 | 결정 |
|---|---|
| 대원칙 | **1:1 금지** (Phase 1~2). 단, **"심사 통과자 추후 허용 예정" 공지** |
| Phase 1 페어 | **FF(여여) / MF(혼성)** — 여성 1명 이상 포함 |
| Phase 2 페어 | ↑ + **GG / BB** — Rainbow Circle (LGBTQ+ 포용) |
| Phase 3 1:1 | `mate_verified` 이상 + 평점 4.8+ + 50회 + 관리자 면접 → 솔로 심사제 |
| 세션 인원 | 메이트 2명 + 게스트 2~4명 = 총 **4~6명** (유료 최대 10명) |
| 세션 길이 | **6시간 (12~18시)** 기본, 관리자 동적 설정 |
| 활동 시간대 | 10~19시 권장. ~~야간·음주·숙박 동행 금지~~ → **2026-04-15 제거: 성인 자율** |
| 페어 파트너 조달 | 본인 데려오기 원칙 + 솔로 대면검증 예외 |
| 솔로 여성 게스트 | 여여(FF) 페어와 우선 매칭 (권장) |
| 한일 커플 | 자연 발생 **허용**, 단 **광고·마케팅 사용 금지** |

> **상세 정책**: ACCESS_CONTROL.md §5 참조

### 6. 개입 모델 — Rails / Recommended / Free 3단
- **Rails (강제)**: 시간·장소·인원·체크인·신원·평가·외부연락 시점
- **Recommended (AI 초안)**: 플랜·맛집·포토·언어·예산 템플릿 → 호스트 자유 수정
- **Free (100% 자율)**: 대화·분위기·즉흥·개인 에피소드

### 7. 경제 모델 (2026-04-16 전면 재설계)
| 항목 | 결정 |
|---|---|
| 플랫폼 번 | **0** (플랫폼은 비용 지급 안 함) |
| **플랫폼 수수료** | **0%** — 매칭·공급자 거래 쉐어 제거 (기존 20%/80% 번복) |
| **플랫폼 수익 원천** | **FestiPoint 팩 판매** (¥500~¥20,000) — 단일 BM |
| 게스트 부담 | 본인 실비 + **메이트 초대비 1/N 분담** (공급자·메이트 직접 수령) |
| 메이트 보상(Phase 1) | 공짜 하루 + 친구 + SNS 자산 + 커리어 레퍼런스 |
| 메이트 숙박비 | 본인 부담 (→ 지역 독점 공급 자연 형성) |
| 유료 상품 자격 | **3회 × 평점 4.5+ 완료 시 Verified 승급** |
| 유료 상품 매출 귀속 | **100% 메이트** (플랫폼 수수료 0%) |
| 유료 상품 인원 상한 | 최대 10명 (무료는 6명) |
| 플랫폼 협상 책임 | 지역예산·티켓확보·식당·숙소·쇼핑 할인 적극 추진 |
| 예외 (법적 별개) | **병원 외국인환자 유치 대가** — 의료관광 별개 사업 범주 (§10) |

> 상세 결제·크레딧 정책: ACCESS_CONTROL.md §4·§7 참조

### 8. 첫 파일럿

### 9. KoJap (한→일 레인)
- **동시 시작, 다른 깊이**: FestiMate는 풀스택 매칭, KoJap은 콘텐츠+AI만
- 파트너의 일본 지역 맛집 100개 칼럼이 즉시 자산화
- 1년 이상 후 KoJap도 매칭 확장 검토
- 수익 모델: 프리미엄 구독 or 제휴 커미션

### 10. 법적 구조 (2026-04-15 일부 번복)
- 회사 등기부에 **외국인환자유치업** 포함 (2026.02.27 추가)
- 의료정보 포털·의료관광사업·의료 컨설팅 등 6개 사업목적 확보
- **외국인환자유치업자 복지부 등록** 필요 (60~90일)
- **보증보험 1억 이상** 가입
- ~~한국인 대상 의료광고 제한 → 언어·지역 기반 UI gating 필요~~ → **2026-04-15 결정으로 번복됨**. 한국인 차단 제거, 변호사 검토 시점에 의료광고법 §56 재확인 (상세: ACCESS_CONTROL.md §8)

## 📚 브레인스토밍 세션 상세 로그

### 2026-04-14 세션 (초기 기획)

#### 세션 흐름
1. 레포 클론 및 현재 상태 분석 (Next.js 16 + Supabase + OpenAI 기반 기존 프로덕트 확인)
2. 브레인스토밍 진입 — 사업 본질 파악
3. 타겟 확정 — 일본 20~30 여성, 한국 인바운드 #1 코호트
4. 공급측 설계 — 관광학과·일어과 대학생 (학교 파트너십)
5. 성별·매칭 구조 논의 → 1:1 금지, 2+ 페어, 최소 1명 여성, MM 금지
6. 개입 수준 논의 → 3단 레일(Rails/Recommended/Free) 모델 확정
7. 경제 모델 논의 → 플랫폼 0원 투입, 게스트가 메이트 비용 분담
8. KoJap 동시 시작 검토 → 콘텐츠·AI만 먼저, 매칭은 나중
9. K-뷰티 DB 추가 발표 → 메인 축으로 격상
10. 포지셔닝 재편 → FestiMate 브랜드 유지, K-뷰티 서브 프로덕트로
11. 4-도시 허브 전략 확정 (서울·부산·청주·제주)
12. 첫 파일럿 결정 → 청주 허브 + 이용강 명장 + 청주대
13. 플랫폼 사고 전환 → 공급자 셀프 업로드 마켓플레이스

#### 기각된 대안들 (왜 안 갔는지 기록)
- **Seoulmate 리브랜딩**: 축제=진짜 해자인데 서울 네이밍이 그 해자를 부정. 지방 동선도 못 살림. FestiMate 유지로 결정.
- **여↔여 한정 모델**: 1:1 금지 룰이 안전을 대부분 해결 → 혼성 허용으로 확장
- **남자 1명 필수 룰**: 차별 논란 + 여학생 공급 절벽 + 시대 역행 → "여성 1 이상 필수"로 대체
- **플랫폼 경비 지원 모델**: 운영 복잡·지속 불가 → 제로 번 + 게스트 분담으로 전환
- **KoJap 동시 풀스택 론칭**: 포커스 분산·마켓플레이스 4축 동시 불가능 → 콘텐츠·AI만 병행
- **Phase 1 유료화**: 신뢰·문화 형성 전 유료화는 역효과 → 3회 무료 후 Verified 승급

#### 주요 통찰 기록
- **"시술 1~2시간 × 축제 반나절"이 여행 동선 DNA** — AI 플래너의 진짜 차별점은 시술 회복 윈도우 + 축제 일정 + 교통 시간 통합 최적화
- **"축제 단위 리퀴디티 응집"** — 매칭 기준을 사람이 아닌 **축제 이벤트 페이지**로 하면 콜드 스타트 쉬워짐
- **"플랜은 AI, 경험은 친구"** — 표준화(품질)와 친밀감(차별)의 교집합
- **"공급자 셀프 업로드가 진짜 플랫폼"** — 상품 30개 만드는 회사가 아니라 상품 10,000개 모이는 플랫폼
- **"매월 크롤링 K-뷰티 DB"** — 의료광고법 회색지대 + 공개 정보 크롤링 합법 → 선점자 절대 우위
