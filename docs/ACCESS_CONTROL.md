## 2. 접근 권한 3단계

GLOSSARY.md §2의 여행자 등급 중 Lv0~2를 접근 제어의 3계층으로 사용.

| 계층 | DB 코드 | 진입 조건 | 핵심 의미 |
|---|---|---|---|
| **Public** | `guest_visitor` | 없음 (비회원) | 누구나 접근. 트래픽 유입 극대화 |
| **Social** | `guest_member` | 소셜 로그인 (Google / Kakao / LINE) | 회원 식별. 개인화 시작 |
| **Verified** | `guest_verified` | 여권 사본 제출 + 관리자 승인 | 신원 확인 완료. 매칭·결제 허용 |

### 2.1. Verified 인증 방식 변경 (2026-04-15 결정으로 번복됨)

> **번복 사항**: MEMBERSHIP_TIERS.md §6에서는 LINE 실명 인증·카카오 실명 인증을 Verified 승급 수단으로 정의했으나, **2026-04-15 결정으로 여권 사본 제출을 유일한 Verified 인증 수단으로 확정**.
>
> - LINE / 카카오 인증은 **Social 계층 진입용** (소셜 로그인)으로 격하
> - Verified 승급에는 **여권 사본 업로드 + 관리자 수동 승인** 필요
> - 사유: 매칭(대면 만남) 안전 보장을 위해 정부 발급 신분증 수준의 인증 필요

### 2.2. 소셜 로그인 공급자

| 공급자 | 주 대상 | OAuth Provider | Phase |
|---|---|---|---|
| **Google** | 글로벌 | Google OAuth 2.0 | Phase 0 |

## 5. 매칭 정책 (2026-04-15 재설계)

### 5.1. 대원칙

- **1:1 매칭 금지** — Phase 1~2에서 공식 정책. 모든 세션은 페어(2명) 이상으로 구성
- 단, **"심사 통과자 추후 허용 예정"을 공지**하여 솔로 활동 희망자 이탈 방지
- Verified (`guest_verified`) 이상만 매칭 요청 가능
- **매칭 성사 수수료 0%** (2026-04-16 확정) — 플랫폼은 AI 심사·연결만. 거래에 개입하지 않음
- **모집 글 작성 월 1회 무료**, 초과 시 5 FP 소모 (§4.3 참조)

### 5.2. Phase별 페어 구성

| Phase | 허용 페어 타입 | 코드 | 설명 |
|---|---|---|---|
| **Phase 1** | 여여 (Female-Female) | `FF` | 안전 최우선. 초기 신뢰 구축 |
| | 혼성 (Male-Female) | `MF` | 여성 1명 이상 포함 |
| **Phase 2** | ↑ + 남남 (Gay-Gay) | `GG` | **Rainbow Circle** — LGBTQ+ 포용 |
| | ↑ + 여여 동성 (Bi-Bi) | `BB` | Rainbow Circle 확장 |
| **Phase 3** | ↑ + **1:1 심사제** | `SOLO` | 심사 기준: `mate_verified` 이상 + 평점 4.8+ + 50회 세션 + 관리자 면접 |

> **기존 PLAYBOOK §5 "MM 페어 금지"는 2026-04-15 결정으로 번복됨.**
> Phase 2에서 GG/BB 페어를 Rainbow Circle로 공식 도입.

### 5.3. 세션 제약 (2026-04-15 변경)

| 항목 | 기존 (PLAYBOOK §5) | 변경 (2026-04-15) |

| 세션 길이 | 6시간 (12~18시) | 유지 (기본값, 관리자 동적 설정) |
| 활동 시간대 | 10~19시만 | 유지 (기본 가드레일, 관리자 동적 설정) |
| **야간 동행 금지** | 금지 | **제거** — 성인 자율. 권고 사항으로만 표시 |
| **음주 동행 금지** | 금지 | **제거** — 성인 자율. 권고 사항으로만 표시 |
| **숙박 동행 금지** | 금지 | **제거** — 성인 자율. 권고 사항으로만 표시 |
| 세션 인원 (무료) | 4~6명 | 유지 |

> **성인 자율 원칙**: 참여자 전원 19세 이상 성인. 야간·음주·숙박은 플랫폼이 금지하지 않으나, **권고 사항**(안전 팁)으로 표시하고 면책 고지를 포함.
>
> 면책 고지 문구 (예): "플랫폼 권장 활동 시간 외 활동에 대해 FestiMate는 책임을 지지 않습니다. 안전을 위해 야간 활동 시 위치 공유를 권장합니다."

### 5.4. 한일 커플 정책

- 메이트-게스트 간 연애 관계 **자연 발생 허용** — 플랫폼이 개입·금지하지 않음
- 단, **광고·마케팅에 로맨스 요소 사용 금지** — "한국 남친/여친 만들기" 등의 표현 절대 불가
- 플랫폼 포지셔닝은 **친구·문화 교류**로 유지
- 이유: 로맨스 마케팅은 안전 이미지 훼손 + 타겟(일본 여성) 이탈 유발

## 6. 운영 체제 — 1인 + AI (2026-04-15 확정)

### 8.1. 전제

- 관리자 1명 (`ik@incutek.net`) + AI 자동화로 전체 플랫폼 운영
- CS 팀 없음, 24/7 긴급 대응 불가
- 이 전제는 모든 설계 문서의 기반 가정

### 8.2. AI 자동 처리 영역

| 영역 | AI 역할 | 사람 개입 시점 |
|---|---|---|
| CS 문의 | 자동 응답 (FAQ + 컨텍스트) | 3회 미해결 시 관리자 에스컬레이션 |
| 번역 | 한/일/영 자동 번역 | 번역 품질 이슈 리포트 시 |
| 상품 검수 | AI 1차 스크리닝 (사진 품질, 금칙어, 중복) | 최종 승인은 관리자 수동 |
| 일정 생성 | AI Guide 전체 자동 | - |
| 매칭 추천 | 알고리즘 자동 | 분쟁 발생 시 관리자 개입 |
| 리뷰 모니터링 | 부정 리뷰·욕설 자동 필터 | 이의 제기 시 관리자 판단 |

### 8.3. 긴급 상황 대응

플랫폼은 **24/7 긴급 대응 불가**. 대신 다음을 제공:

| 상황 | 플랫폼 제공 | 안내 대상 |
|---|---|---|
| 사건·사고 | 긴급 버튼 → 경찰 112, 소방 119 | 현지 기관 |
| 의료 응급 | 긴급 버튼 → 119, 가까운 병원 목록 | 현지 기관 |
| 분쟁·불만 | AI 챗봇 → 관리자 에스컬레이션 (영업시간 내) | 관리자 |
| 성범죄·폭력 | 긴급 버튼 → 경찰 112 + 자동 세션 종료 | 현지 기관 + 플랫폼 |

- 긴급 버튼은 **모든 세션 진행 중 화면 상단 고정**
- 버튼 클릭 시: (1) 현지 기관 연결 정보 표시, (2) 상대방에게 알림, (3) 관리자에게 로그 전송
- 관리자 응답 시간: 영업시간(09~18 KST) 내 **4시간 이내**

## 7. 결제 로드맵 (2026-04-16 전면 재작성)

### 7.1. 대원칙 — 플랫폼 결제 1종

- **플랫폼이 받는 결제는 FestiPoint 팩 구매 1종 뿐** (§4.4)
- 상품·시술·숙소·식당 결제는 **공급자가 직접 수령** (플랫폼 미개입)
- **수수료 0%** — 매칭·공급자 거래에서 쉐어 받지 않음
- 책임 분리: 플랫폼은 "AI 심사·연결"만 수행

### 7.2. 공급자 결제 구조

| 카테고리 | 결제 흐름 | 플랫폼 역할 | 비고 |
|---|---|---|---|
| `master` (명인 체험·쇼핑) | 여행자 → **공급자 직접 수령** | AI 심사·노출 | 계좌이체·페이팔·PayPay 등 공급자 선택 |
| `templestay` | 여행자 → **사찰 직접 수령** | AI 심사·노출 | 사찰 예약 채널 안내 |
| `beauty` (병원 시술) | 여행자 → **병원 직접 수령** | AI 심사·노출·안내 | 외국인환자유치업 등록 후 연결 (별개 사업 범주) |
| `sleep` | 여행자 → 외부 플랫폼 (Agoda 등) | 어필리에이트 링크만 | 플랫폼 커미션 3~4% (외부 수령) |
| `food` | 여행자 → 식당 직접 | 링크·예약 연결 | 제휴 시 커미션 (외부 수령) |
| `mate_session` | 게스트 → **메이트 직접** (1/N 분담) | AI 심사·페어 승인 | 플랫폼 수수료 0% |

### 7.3. 플랫폼 결제 수단 (FestiPoint 팩 구매용)

오픈 시점:
- PG: **토스페이먼츠** (한국 PG, 일본 카드·PayPay·LINE Pay 지원)
- 결제 통화: JPY 기준 가격 표시, 원화 환산 결제 가능
- 영수증: 이메일 자동 발송

### 7.4. 수수료 정책

**플랫폼이 받는 수수료 = 0%.** (2026-04-16 확정, 기존 PLAYBOOK §7 "유료 상품 수수료 20%"는 번복됨)

예외:
- **병원 외국인환자 유치 수수료**: 외국인환자유치업 법 테두리 내 별개 사업 범주. 수수료가 아닌 "유치 대가"로 포지셔닝 (PLAYBOOK §10, LEGAL_CHECKLIST #14)
- **외부 어필리에이트**: Agoda/Booking 등 외부 플랫폼이 FestiMate에 지급하는 커미션. 공급자·여행자 부담 없음

### 7.5. 수익 원천 (2027+ 단순화)

## 8. K-뷰티 한국인 차단 정책 (2026-04-15 결정으로 번복됨)

> **번복 사항**: PLAYBOOK §10 및 SITEMAP_SPEC §6에서는 K-뷰티(beauty) 카테고리에 대해 한국인 접근을 IP·언어·로그인·KoJap 전환의 4단계로 차단하도록 정의했으나, **2026-04-15 결정으로 한국인 차단 기능을 제거**.

### 8.1. 번복 내용

| 항목 | 기존 (PLAYBOOK §10 / SITEMAP §6) | 변경 (2026-04-15) |
|---|---|---|
| 한국 IP 차단 | `/beauty` 403 차단 | **제거** — 한국인도 열람 가능 |
| 언어 UI 숨김 | `lang=kr` 시 KBeauty 네비 숨김 | **제거** — 모든 언어에서 노출 |
| 로그인 국적 확인 | 국적 기반 이중 차단 | **제거** |
| KoJap 자동 전환 | 한국인에게 KoJap 추천 | 유지 (KoJap 홍보 차원, 차단 목적 아님) |

### 8.2. 변호사 검토 재확인 필요

한국인 차단 제거에 따라 아래 법적 사항을 **변호사 검토 시점에 재확인**:

| 검토 항목 | 우려 사항 | 확인 시점 |
|---|---|---|
| **의료광고법 제56조** | 한국인 대상 의료광고 규제. 차단 없이 가격·시술 정보를 한국어로 노출 시 의료광고에 해당하는지 | Phase 0 변호사 검토 시 |
| **외국인환자유치업 범위** | 유치업 등록 후 한국인에게도 정보를 제공하는 것이 등록 범위 내인지 | 복지부 등록 신청 시 |
| **가격 표시 제한** | 한국인에게 시술 가격을 직접 노출해도 되는지, "문의" 버튼으로 대체해야 하는지 | Phase 0 변호사 검토 시 |

### 8.3. 변호사 검토 결과에 따른 분기

```
변호사 판단 A: 한국인 노출 OK
  → 현행 유지 (차단 없음)

변호사 판단 B: 가격만 제한
  → 한국 IP / lang=kr 시 가격 숨김 + "문의하기" 버튼 대체
  → 시술 정보·병원 정보는 노출 가능

변호사 판단 C: 전면 차단 필요
  → SITEMAP_SPEC §6의 기존 4단계 차단 복원
  → 이 문서 §6 재개정
```

- **현재 기본값**: 판단 A (차단 없음)로 구현
- 분기 B/C는 관리자 페이지에서 플래그 전환으로 대응 가능하도록 설계 (코드 배포 없이)

### 8.4. PLAYBOOK §10 / SITEMAP §6과의 관계

| 기존 문서 | 기존 내용 | 현재 상태 |
|---|---|---|
| PLAYBOOK §10 | "한국인 대상 의료광고 제한 → 언어·지역 기반 UI gating 필요" | **2026-04-15 결정으로 번복됨**. 차단 제거, 변호사 검토 시 재확인 |
