## 1. 핵심 철학 (The One Sentence)

> **"AI는 사람을 대체할 수 없다. 그렇지만 Mate가 구해지지 않으면 AI랑 여행하고, 구해져도 AI는 항상 불러서 써야 한다."**

— 사용자, 2026-04-19

이 문장이 제품 설계의 북극성. 모든 기능·UI·마케팅·가격 책정이 이 문장과 정합해야 함.

## 2. 3대 원칙

### 원칙 1: 🪓 AI ≠ 사람 (대체 금지)
- AI는 Mate 흉내 내지 않음 (체온·즉흥·현지 감성은 Mate 고유)
- AI는 본인이 AI임을 숨기지 않음
- "당신의 친구"가 아니라 "당신의 도우미"
- 페르소나는 있되 의인화는 절제

### 원칙 2: ⏰ AI = 상시, Mate = 세션
- **AI**: 여행 전·중·후 24/7 접근 가능
- **Mate**: 6시간 세션 (12~18시) 기본, 필요시 연장 — PLAYBOOK §5 준수

### 원칙 3: 🔗 공존 가능, 대체 불가
- AI가 편리(logistics)를, Mate가 경험(heart)을 제공
- 둘 다 있을 때 최고의 여행
- AI 단독도 기본 서비스는 성립 (Mate 공급 부족 완화)

## 3. 3가지 역할 정의

### Role 1: PLAN 모드 — 여행 Planner (사전)

| 항목 | 내용 |
|------|------|
| 시점 | 여행 결정 전 ~ 예약 완료 |
| 입력 | 여행 날짜·기간·관심사·예산·인원·출발지 |
| 산출 | Day-by-Day 여행 플랜 (PLAN_TEMPLATE_SCHEMA 준수) |
| 제약 | 시술 회복 윈도우, 축제 일정, 교통시간, 체크인 시간 |
| 언어 | 일본어 (주), 영어 (부) |
| 구현 | `/ai-guide` 라우트, OpenAI `gpt-4o-mini` + RAG + 템플릿 |
| 상태 | ✅ 스캐폴드 완료, Mate 가용성 인지 기능 미구현 |

**추가 필요 기능**:
- Mate 가용성 쿼리 → 있으면 플랜에 통합, 없으면 "AI 단독 여행 모드" 옵션 제시
- 플랜 저장 + 공유 URL (`/jp/ai-guide/[plan-id]`)

### Role 2: MATCH 모드 — Matching Guide (매칭 단계)

| 항목 | 내용 |
|------|------|
| 시점 | 플랜 확정 후 ~ 여행 출발 전 |
| 입력 | 플랜 + 선호 (성별·연령·언어·취향·예산 분담) |
| 산출 | Mate 페어 추천 리스트 + 매칭 성사 가이드 |
| 기능 | ① 추천 (지역·시기·언어·성향 매칭) ② 아이스브레이커 제안 ③ 사전 브리핑 |
| 언어 | 일본어 (게스트측) + 한국어 (메이트측 코칭용) |
| 구현 | `/mate` 라우트 스캐폴드, 지능형 매칭 미구현 |
| 상태 | 🟡 UI만 있고 로직 없음 |

**추가 필요 기능**:
- Mate 프로필 벡터화 (pgvector 활용)
- 게스트 선호 ↔ Mate 특성 유사도 계산 (**LLM 아닌 알고리즘**)
- 매칭 결과 설명 ("이 메이트를 추천하는 이유", `mini`로 포맷팅)
- 첫 메시지 초안 생성 (템플릿 + 번역, `mini` 충분)

### Role 3: TRIP 모드 — Travel Assistant (여행 중)

| 항목 | 내용 |
|------|------|
| 시점 | 여행 출발 후 ~ 종료 |
| 입력 | 현재 위치·시간·플랜 진행 상황·사용자 질문·사진 |
| 산출 | 실시간 안내·번역·추천 조정·긴급 대응 |
| 모드 A (Mate 있음) | AI = 백엔드 지원 (번역·정보·플랜 조정), Mate = 전면 대화 |
| 모드 B (Mate 없음) | AI = 전면 도우미, 사용자의 유일한 동반자 |
| 언어 | 일본어·영어 (게스트 대면), 한국어 (정보 원천) |
| 구현 | ❌ **미구현** (M3 2026-07월 예정) |

**추가 필요 기능 (신규)**:
- 모바일 퍼스트 실시간 인터페이스
- 위치 기반 추천 (현재 위치 → 근처 볼거리·맛집)
- 카메라 입력 ("이 간판/음식이 뭐에요?")
- 실시간 번역 (음성·이미지) — `mini vision` 또는 전용 번역 API 비교
- 플랜 진행 상황 기반 다음 안내
- 긴급 프로토콜 (112/119 연결 안내, Mate·관리자 자동 알림)
- 여행 기록 (사진·이동·소비 자동 정리)
- **유일하게 고급 추론 필요 케이스**: 여행 중 플랜 재구성 (예상 밖 상황 → 일정 재계산)

## 4. 4대 설계 결정

### 4-1. 인격 연속성 (Memory)
**단일 페르소나 "AI 여행 도우미"** — 플래너·매처·도우미 세 역할을 같은 AI가 담당.

- 사용자 기억·대화 히스토리 연속 유지
- 여행 3개월 전 대화가 여행 중에도 이어짐
- DB: `user_ai_memory` 테이블 (userId, conversation_history, preferences, trip_context)
- 기술: LangChain/LangGraph 또는 자체 컨텍스트 관리

### 4-2. BM 정합
**FestiPoint 단일 재화 유지** (PLAYBOOK §7 준수)

| 사용 | 비용 |
|------|------|
| AI 기본 채팅 (텍스트) | 무료 |
| AI 고급 기능 (이미지 분석·실시간 번역·음성) | 포인트 차감 |
| Mate 세션 | 포인트 차감 |
| AI 플랜 저장·공유 | 무료 |
| AI 저장된 플랜 수 제한 | 무료: 3개 / Verified: 무제한 |

> **⚠️ 구체 수치는 이 문서에 두지 않음.** 모두 `ADMIN_CONFIGURABLE_PARAMS.md` §3-1 (가격·재화) · §3-4 (AI 사용량) 참조. 관리자 페이지에서 실시간 조정.

### 4-3. Handoff (역할 전환 UX)
**사용자는 하나의 앱만 알고, 내부에서 자동 전환**

- 탭/메뉴 분리 없음
- 여행 단계에 따라 AI가 자동 모드 전환
- UX 예시:
  - "여행을 계획 중이신가요?" → PLAN 모드
  - "메이트 찾아볼까요?" → MATCH 모드
  - "지금 여행 중이신가요?" → TRIP 모드
- 전환 트리거: 여행 날짜·위치·사용자 명시적 선택

### 4-4. Mate-Side AI
**Mate 포털(`/mate-portal`)에 별도 AI 보조 기능**

| 시점 | 기능 |
|------|------|
| 세션 전 | 게스트 프로필 요약, 추천 대화 주제, 관심사 기반 장소 추천 |
| 세션 중 | 통역 보조, 위치 기반 정보 검색, 긴급 상황 대응 |
| 세션 후 | 리뷰 작성 도움, 피드백 구조화, 세션 로그 정리 |

→ Mate도 AI 혜택을 누림으로써 공급측 매력도 ↑

## 5. Mate와의 경계 (Boundary)

| Mate 고유 영역 (AI 불가) | AI 영역 (Mate 있어도 AI가 담당) |
|--------------------------|--------------------------------|
| 현지인 감성·체온·친근함 | 모든 정보 검색·번역 |
| 즉흥적 현장 판단 | 플랜 재계산·경로 최적화 |
| 예약·결제 대신 실행 | 메뉴·간판·안내판 번역 |
| 사진 찍어주기·동반 | 문화·역사 설명 |
| "로컬만 아는 곳" 즉석 소개 | 축제 일정·시간 알림 |
| 1인 푸대접 해결 (같이 먹어줌) | 장소 추천·길찾기 |
| 비언어적 소통 (표정·제스처) | 기록·사진 분류·일정 조정 |

## 7. 기술적 함의 (구현 로드맵)

### 7-0. 최상위 원칙 — "LLM 최소·구조화 먼저" ⭐

FestiMate의 AI 유스케이스 **거의 전부가 RAG·검색·템플릿**입니다. 고급 LLM 추론이 필요한 케이스는 극소수 (여행 중 즉석 리플랜 정도).

**기본 원칙**:
- LLM을 쓰기 전에 **알고리즘·벡터검색·템플릿**으로 풀 수 있는지 먼저 확인
- LLM을 써야 하면 **`gpt-4o-mini` 기본**, 품질 문제 입증 시에만 상향
- 모델 선택은 `ADMIN_CONFIGURABLE_PARAMS.md` §3-4에서 config로 관리 (배포 없이 교체 가능)

| 유스케이스 | 실제 작동 방식 | 필요 모델 |
|-----------|-------------|----------|
| 여행 플랜 생성 | DB retrieval + 시간·이동 제약 알고리즘 + 일정 템플릿 | `mini` + 구조화 |
| 채팅 Q&A | 검색 + 자연어 포맷팅 | `mini` |
| 번역 JP↔KR↔EN | 번역 | `mini` (출시 후 DeepL·Gemini Flash 비교) |
| 이미지 번역 (OCR) | OCR + 번역 | `mini vision` |
| 메이트 매칭 | 벡터 유사도 (pgvector) | **LLM 불필요** |
| 첫 메시지 초안 | 템플릿 + 변수 치환 | 템플릿 + 번역 |
| 여행 중 리플랜 | 제약 재충족 + 대안 탐색 | `mini` (유일하게 고급 추론 케이스) |

**비용 함의 (1K MAU 기준)**: 약 $6/월. 10K MAU에서도 약 $60/월. 1년 운영 여유.

### 7-1. 구현 로드맵

| 역할 | 현재 | M3 (2026-07월) | M5 (2026-09월) | M7 (파일럿) |
|------|------|---------------|---------------|------------|
| **PLAN** | 기본 채팅 | + Mate 가용성 인지 | + 플랜 저장·공유 | + 실제 유저 A/B |
| **MATCH** | UI만 | 벡터 매칭 (LLM 없이) | + 아이스브레이커 | + 매칭 성사율 측정 |
| **TRIP** | 없음 | 기본 챗봇 모바일 | + 위치·카메라·음성 | + 긴급 프로토콜 |
| **Mate AI** | 없음 | Mate 포털 통합 | + 통역 보조 | + 세션 로그 자동 |

## 9. 미결 사항 (Future Decisions)

이 문서에서 지금 결정하지 않은 것들 — 추후 세션에서:

1. **AI 페르소나 이름 부여 여부** (예: "하나", "사쿠라", "없음" 중립)
2. **음성 입출력 (TTS/STT) 포함 여부** — 모바일 경험에 중요
3. **오프라인 지원 범위** — 지하철·산간 신호 약한 지역
4. **AI가 배포하는 정보의 법적 면책 범위** — "AI 추천은 참고만" 고지 수준
5. **Mate가 거부한 매칭 요청에 AI가 대안 자동 제시** 여부
6. **번역 엔진 선택** — `mini` vs DeepL vs Gemini Flash (출시 후 품질·비용 비교)
