# 1.5단계: 사전 온라인 상담 설계 (CONSULTATION_PAGE_DESIGN.md)

> **버전**: v0.1
> **작성**: 2026-04-20
> **상태**: 기획 확정
> **환경**: Claude Desktop (기획 전담)

> 📌 **로컬 규칙**: 모든 화면은 한국어 버전만 작성. 구현 시 일본어 추가.

---

## 개요

**1단계** (검색·발견·AI 플래너)와 **2단계** (방문·시술) 사이의 연결고리.

일본인 관광객이 시술을 결정한 뒤, 실제 방문 전에 클리닉과 사전 소통하는 흐름.

| 유저 문제 | FestiMate 해결 |
|----------|--------------|
| 클리닉에 뭘 물어봐야 할지 모름 | 3가지 핵심 질문 자동 생성 |
| 언어 장벽 (한국어·일본어) | 템플릿 자동 작성, 구현 시 일본어 번역 |
| 어느 채널로 연락해야 하는지 모름 | LINE / 카카오 채널 선택 UI |
| 문의 후 상태 관리 어려움 | 문의함 상태 추적 |

---

## 1. 서비스 범위 (Phase 1 기준)

```
Phase 1 (지금):
  유저 → 문의 템플릿 생성 → LINE/카카오 딥링크 발송
  클리닉 → LINE/카카오로 직접 답변
  FestiMate → 발송 연결 + 상태 추적 UI 제공

Phase 2 (이후):
  FestiMate 내부 메시지함
  AI 번역 중계 (일본어 ↔ 한국어)
  자동 예약 연결
```

**Phase 1에서 FestiMate는 중개 플랫폼이 아닌 연결 도구다.**
클리닉과의 실제 상담·계약은 유저 책임.

---

## 2. 진입점

문의하기 버튼은 두 곳에서 진입 가능:

```
A) 클리닉 상세 페이지 (/kr/beauty/clinics/[id])
   → [문의하기] 버튼

B) AI 뷰티 플래너 결과 페이지 (/kr/beauty/planner/result/[id])
   → 추천 클리닉 카드의 [문의하기] 버튼
   → AI 플래너 데이터 (시술·귀국일·특이사항) 자동 주입
```

B 경로로 진입하면 Step 2 조건 입력이 자동으로 채워진다.

## 3. 문의 생성 플로우 (4단계)

### Step 1: 시술 선택

```
┌─────────────────────────────────────────────────┐
│  강남 OO피부과에 문의하기                         │
│                                                 │
│  어떤 시술을 문의할까요?                          │
│                                                 │
│  ☑ 물광주사        ← AI 플랜에서 담은 시술 자동표시│
│  ☑ 보톡스 (턱)                                  │
│  □ 레이저토닝                                    │
│  □ 직접 입력...                                  │
│                                                 │
│  이 클리닉이 제공하는 시술만 표시됩니다.           │
│                                    [다음 →]     │
└─────────────────────────────────────────────────┘
```

- AI 플래너에서 담은 시술 → 자동 체크
- 클리닉이 제공하지 않는 시술은 목록에서 제외
- 직접 입력 옵션 제공 (AI 플래너 미사용 유저용)

---

### Step 2: 내 조건 확인

```
┌─────────────────────────────────────────────────┐
│  내 조건 확인                                    │
│  (클리닉이 적합 여부 판단에 사용합니다)            │
│                                                 │
│  귀국 예정일                                     │
│  ┌─────────────────────┐                        │
│  │  2026년 5월 3일  📅  │  ← AI 플래너 자동 입력  │
│  └─────────────────────┘                        │
│                                                 │
│  희망 방문 날짜                                   │
│  ┌─────────────────────┐                        │
│  │  2026년 4월 30일 📅  │                        │
│  └─────────────────────┘                        │
│                                                 │
│  특이사항 (의사가 꼭 알아야 할 것)                 │
│  ┌─────────────────────────────────────────┐    │
│  │ 예: 임신 중 / 복용약 있음 / 켈로이드 체질  │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  예산 범위 (선택)                                 │
│  [~50만원]  [50~100만원]  [100만원+]  [미정]     │
│                                                 │
│  ⚠ 입력 내용은 클리닉 문의에만 사용됩니다.        │
│                           [다음 →]              │
└─────────────────────────────────────────────────┘
```

- AI 플래너 경로: 귀국일·특이사항 자동 입력
- 직접 진입: 모두 수동 입력
- 특이사항은 BEAUTY_PAGE_DESIGN §0 Disclaimer 원칙 적용

```
│  AI가 자동으로 작성했습니다. 수정 가능합니다.       │
│                                                 │
│  ┌─────────────────────────────────────────┐   │
│  │ 안녕하세요.                               │   │
│  │ FestiMate를 통해 문의드립니다.             │   │
│  │                                         │   │
│  │ [희망 시술]                               │   │
│  │ • 물광주사                               │   │
│  │ • 보톡스 (턱)                            │   │
│  │                                         │   │
│  │ [귀국 예정일] 5월 3일                    │   │
│  │ [방문 희망일] 4월 30일                   │   │
│  │ [특이사항] 없음                          │   │
│  │ [예산] 50만원 내외                       │   │
│  │                                         │   │
│  │ 아래 사항을 확인해 주시겠어요?             │   │
```

**자동 생성 3가지 질문 (고정):**
1. 시술 가능 여부
2. 정확한 가격
3. 방문 전 준비사항

구현 시 일본어로 자동 번역 후 발송.

### Step 4: 채널 선택 + Disclaimer 동의

**채널 커버리지 (확정):**
- **카카오채널**: 한국 모든 클리닉 보유 → 100% 커버. 기본 채널.
- **LINE**: 일본인 타겟 클리닉 보유 → 추가 옵션.
- 두 채널 모두 문의 가능. 유저가 선택.

```
┌─────────────────────────────────────────────────┐
│  어떤 채널로 보낼까요?                             │
│                                                 │
│  ┌──────────────────┐   ┌──────────────────┐   │
│  │  💛 카카오로 보내기│   │  💚 LINE으로 보내기│   │
│  │                  │   │                  │   │
│  │  ✅ 모든 클리닉   │   │  이 클리닉 등록됨  │   │
│  └──────────────────┘   └──────────────────┘   │
│                                                 │
│  카카오: 모든 클리닉 이용 가능                     │
│  LINE: 등록된 클리닉만 활성화                      │
│                                                 │
│  ─────────────────────────────────────────────  │
│                                                 │
│  보내기 전 확인해 주세요                           │
│                                                 │
│  ☑ FestiMate는 예약·상담을 대행하지 않습니다.     │
│    클리닉과의 상담·계약은 본인 책임입니다.          │
│    시술 결정은 의사의 직접 진단 후 이루어져야       │
│    합니다.                        (필수 동의)    │
│                                                 │
│                         [동의 후 발송하기]        │
└─────────────────────────────────────────────────┘
```

**채널 활성화 규칙:**
- 카카오채널: 한국 모든 클리닉 보유. 항상 활성. (기본)
- LINE: 클리닉 DB에 `line_id` 있는 경우만 활성. (추가 옵션)
- 카카오 없는 클리닉은 FestiMate 등록 불가 (공급자 등록 필수 항목)

```
│  (방문 당일 일정 관리에 사용됩니다)                 │
│                                                 │

│                                                 │

│                                                 │
│  확정 가격                                       │
│  ┌──────────────┐                              │
│  │  450,000원   │                              │
```


**상태 흐름 (유저가 수동 업데이트):**

```
📨 발송완료 → 📬 답변받음 → 📅 예약완료 → ✅ 시술완료 → ⭐ 리뷰쓰기
```

| 상태 | 아이콘 | 다음 액션 |
|------|--------|---------|
| 발송완료 | 📨 | 답변 받으면 상태 변경 |
| 답변받음 | 📬 | 예약 확정 시 상태 변경 |
| 예약완료 | 📅 | 시술 후 완료 처리 |
| 시술완료 | ✅ | 리뷰 작성 유도 |
| 리뷰완료 | ⭐ | 종료 |

## 6. DB 연결

기존 `clinic_inquiries` 테이블 (KBEAUTY_DB_SCHEMA.md §7) 사용.

```sql
-- 문의 생성 시 저장
INSERT INTO clinic_inquiries (
  user_id,
  clinic_id,
  procedure_ids,      -- Step 1 선택 시술 목록
  departure_date,     -- Step 2 귀국일
  visit_preferred_date, -- Step 2 방문 희망일
  special_notes,      -- Step 2 특이사항
  budget_range,       -- Step 2 예산
  message_body,       -- Step 3 템플릿 내용
  channel,            -- Step 4 'line' | 'kakao'
  status              -- 'sent'
)
```

**상태 업데이트:**
```sql
UPDATE clinic_inquiries
SET status = 'responded'  -- 유저가 "답변 받았어요" 클릭 시
WHERE id = ?
```

## 8. Phase 2 확장 계획

| 기능 | Phase 1 | Phase 2 |
|------|---------|---------|
| 발송 방법 | LINE/카카오 딥링크 | FestiMate 내부 메시지함 |
| 번역 | 구현 시 자동 번역 후 발송 | AI 실시간 번역 중계 |
| 상태 추적 | 유저 수동 업데이트 | 클리닉 답변 자동 감지 |
| 예약 연결 | 없음 | FestiMate 직접 예약 |
| 클리닉 대응 | LINE/카카오 기존 채널 | FestiMate 클리닉 대시보드 |
