### 7. 접근 권한·결제·정책 문서 작성 — ACCESS_CONTROL.md

기존 설계 문서에 흩어져 있던 접근 권한·결제·K-뷰티 정책을 단일 문서로 통합 정의.

#### 주요 결정 사항

- **대원칙 5개 확정**: 트래픽→회원→인증→유료 퍼널, 1년 무수익 OK, 비용 최소화, AI 자동 응대, 관리자 동적 설정
- **접근 권한 3단계**: Public(비회원) → Social(소셜로그인) → Verified(여권제출)
- **Verified 인증 방식 변경**: LINE/카카오 실명 인증을 Social 계층으로 격하, 여권 사본 제출을 유일한 Verified 수단으로 확정 (MEMBERSHIP_TIERS §6 번복)
- **소셜 로그인**: Google / Kakao / LINE 3개 동시 구현 (Phase 0)
- **관리자**: ik@incutek.net

#### 결제 로드맵

오픈 시 결제 없음 → 명인 쇼핑 → 명인 체험 → 템플스테이 → 병원 → 숙소 → 식당 순차 도입.
`master`/`templestay`는 포인트 결제 허용 (상한 30%), `beauty`는 PG 전액 필수.

#### K-뷰티 한국인 차단 제거 (번복)

- PLAYBOOK §10 "언어·지역 기반 UI gating 필요" → **2026-04-15 결정으로 번복됨**
- SITEMAP_SPEC §6 "4단계 다층 방어" → **2026-04-15 결정으로 번복됨**
- 변호사 검토 시점에 의료광고법 §56, 외국인환자유치업 범위, 가격 표시 제한 재확인 필요
