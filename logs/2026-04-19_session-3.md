### 2-2. AI Planner = 양쪽 공용 Canvas (★ 핵심 설계)

사용자 통찰:
> "planner 페이지에서 matching에 Yes 하면 매칭 메뉴로 이어진다."

**구조**:

- **같은 UI·같은 AI·다른 의도**. DRY 원칙 정점.

### 2-3. AI Matching = 수직 분할 리스트

```
좌: Tourist 요청 리스트  │  우: Local Mate 제안 리스트
```

### 2-4. 3-Tier 정보 공개 모델

| Tier | 공개 조건 | 예시 필드 |
|------|---------|---------|
| **Tier 1** | 무조건 공개 | 성별, 연령대, 지역, 일정, 인원, 관심사, 예산대, 언어, 한 줄 소개 |
| **Tier 2** | 상호 ❤️ 해제 (무료) | 자기소개 상세, 평점, SNS, 시간대, 페어 포맷 |
| **Tier 3** | 예약 확정 + 포인트 차감 | 얼굴 사진, 본명, 연락처, 학적 |

**법적 함의**: Tier 3는 민감정보 유사, 별도 동의·파기 기한 필요.

### 2-8. Offering 2축 모델

```
Applicant:   Offering 없음
```

### 2-9. 매칭 승인 = 쌍방 ❤️

### 2-10. 쌍방향 이니시에이팅 허용

Tourist·Mate 어느 쪽이든 먼저 관심·제안 가능. 매칭 페이지 수직 분할 구조와 정합.

### 2-11. 5단계 매칭 플로우

```
1. 탐색         (무료, Tier 1)
2. 관심 표시     (상호 ❤️ → Tier 2 해제, 메시징 활성)
3. 예약 확정     (포인트 차감 → Tier 3 해제)
4. 세션 진행     (체크인 → AI Guide → 체크아웃)
5. 리뷰         (48h 내 필수, 양쪽 서로)
```

**취소·환불 정책** (ADMIN_CONFIG §3-11):
- 72h 전 100% / 24~72h 50% / 24h 미만 0%
- Mate 취소 → Tourist 전액 환불 + Mate 평점 감점

**No-show**:
- Mate: Tourist 환불 + Mate 경고 (3회 자격 정지)
- Tourist: 환불 없음 + 경고
